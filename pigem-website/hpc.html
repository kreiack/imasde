Estrategias de Paralelización

Social Force Model - GPU CUDA

Thread Mapping: 1 thread = 1 agente
Memory Pattern: Coalesced access
Shared Memory: Neighbor cache
Occupancy Target: 75%
2048 threads/SM
900 GB/s Ancho banda
50x-100x Speedup

Pathfinding - CPU Paralelo

Algoritmo: A* Jerárquico (HPA*)
Paralelización: Por agente/cluster
Optimización: Cache de rutas
Balanceo: Por región geográfica
Nivel 0: Celdas (1m²) - A* clásico
Nivel 1: Clusters (100m²) - A* abstracto
Nivel 2: Sectores (1km²) - Dijkstra

DES - Actor Model

Event Queue: Particionado por tiempo
Sincronización: Barriers periódicos
State Updates: Copy-on-Write
Consistencia: MVCC + Eventual
100K eventos/sec
50K ops/sec
